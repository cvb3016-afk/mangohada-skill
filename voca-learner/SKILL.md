# 업무 용어 학습 관리 스킬

노션에 용어를 입력해두면 매일 카카오톡으로 퀴즈를 보내는 간격 반복 학습 시스템입니다. 퀴즈가 발송될 때마다 복습 간격이 자동으로 늘어나며, 별도 응답이 필요 없습니다.

## 사용 가능한 MCP 도구

- **Notion**: `notion-search`, `notion-create-database`, `notion-create-pages`, `notion-update-page`, `notion-fetch`
- **카카오톡**: `KakaotalkChat-MemoChat` (나에게 보내기)
- **스케줄**: `mcp__scheduled-tasks__create_scheduled_task`, `mcp__scheduled-tasks__list_scheduled_tasks`

> ⚠️ `notion-query-database-view`는 플랜에 따라 사용 불가한 경우가 있습니다. 용어 조회는 항상 `notion-search`로 데이터베이스를 찾은 뒤 `notion-fetch`로 내용을 가져오는 방식을 기본으로 사용하세요.

---

## 노션 데이터베이스 구조

스킬이 사용하는 데이터베이스 이름: **"업무 용어 학습 단어장"**

| 속성명 | Notion 타입 | 설명 |
|--------|------------|------|
| 용어 | title | 저장할 용어 |
| 원어 | rich_text | 약어의 풀네임 또는 원문 명칭 |
| 한 줄 정의 | rich_text | 한 문장 요약 |
| 상세 설명 | rich_text | 실무 관점 상세 설명 |
| 활용 분야 | multi_select | UX/UI, BX/브랜딩, 마케팅, PM/PO, 개발, 데이터, 비즈니스, 리서치, HR, 기타 |
| 활용 상황 | rich_text | 실제 사용 맥락 |
| 실무 예시 | rich_text | 업무 상황 예문 |
| 연관 용어 | rich_text | 함께 알면 좋은 용어 최대 3개 |
| 난이도 | select | 초급 / 중급 / 고급 |
| 최초 등록일 | date | 자동 기록 |
| 마지막 복습일 | date | 퀴즈 발송 시 자동 갱신 |
| 복습 횟수 | number | 기본값 0 |
| 숙지 상태 | select | 미숙지 / 복습 중 |
| 다음 복습 예정일 | date | 퀴즈 발송 시 자동 계산 |

## 시나리오별 동작

### 1. 초기 설정 (처음 한 번만)

"데이터베이스 만들어줘" 또는 처음 스킬을 사용할 때:

1. `notion-create-database`로 위 속성을 모두 포함한 데이터베이스 생성
2. 데이터베이스 URL 안내
3. 복습 스케줄(매일 8:30) 등록

이후부터 사용자는 노션 데이터베이스에 직접 용어를 추가하거나, Claude에게 말하면 자동으로 저장됩니다.

---

### 2. 용어 뜻 질문 → 답변 + 자동 저장

사용자가 "~가 뭐야?", "~이 뭔지 알려줘", "~이 무슨 뜻이야?" 같이 업무 용어의 뜻을 물어보면:

**① 먼저 답변**
실무 초급자 눈높이로 용어를 설명합니다. 한 줄 정의 → 상세 설명 → 실무 예시 순으로 자연스러운 문장으로 답합니다.

**② 노션에 자동 저장**
답변 직후 `notion-search`로 중복 확인 후 `notion-create-pages`로 노션에 저장합니다.

저장 완료 시 아래 한 줄을 덧붙입니다:
```
📌 노션 단어장에 저장했어요. 내일부터 복습 퀴즈에 포함됩니다!
```

---

### 3. 용어 직접 저장

사용자가 용어·약어·개념을 말하거나 "저장해줘"라고 하면:

**① 데이터베이스 찾기**
`notion-search`로 "업무 용어 학습 단어장" 데이터베이스를 검색합니다. 없으면 새로 만들지 물어봅니다.
데이터베이스를 찾으면 `notion-fetch`로 속성 목록을 확인합니다. "다음 복습 예정일" 속성이 없으면, 저장 전에 사용자에게 알리고 `notion-update-data-source`(또는 `notion-create-database` 재생성 제안)로 추가할 수 있는지 확인하세요. 추가가 어려우면 해당 필드 없이 진행합니다.

**② 중복 확인**
`notion-search`로 동일 용어가 있는지 확인합니다. 있으면 `notion-fetch`로 내용을 가져와 보여주고 업데이트 여부를 묻습니다.

**③ 내용 자동 작성 원칙**
- 실무 초급자가 이해할 수 있는 수준으로 작성
- UX/UI·디지털 프로덕트 관점 우선
- 추상적 정의보다 실제 업무 예시 중심
- 어려운 개념은 비유로 풀어냄

**④ 저장 및 확인**
`notion-create-pages`로 저장 후 아래 형식으로 확인 메시지를 출력합니다:

```
✅ 'IDI' 저장 완료!

용어: IDI
원어: In-depth Interview
한 줄 정의: 사용자 한 명을 깊이 인터뷰하는 질적 리서치 방법
활용 분야: UX/UI, 리서치
난이도: 중급
연관 용어: FGI, Survey, Usability Test
다음 복습 예정일: [내일 날짜]
```

---

### 4. 복습 퀴즈 발송 ("오늘 복습" / 매일 8:30 자동 발송)

`notion-search`로 "업무 용어 학습 단어장"을 찾고, `notion-fetch`로 저장된 용어 목록을 가져옵니다. 숙지 완료 용어를 제외하고 **우선순위에 따라 3개**를 선정합니다.

**선정 우선순위 (숙지 완료 용어는 제외):**
1. 다음 복습 예정일이 오늘 이전인 용어 (기한 초과)
2. 다음 복습 예정일이 오늘인 용어
3. 복습 횟수가 적은 용어
4. 최초 등록일이 오래된 용어

**발송 방식 — 카카오톡 2개 메시지만 발송 (질문 3개 묶음 → 정답 3개 묶음):**

**메시지 1 — 질문 3개 묶음:**
```
━━━━━━━━━━━━━━━━━━
📚 오늘의 업무 용어 복습
━━━━━━━━━━━━━━━━━━

Q1. [용어1]
원어: [원어1] | 분야: [활용 분야1]

Q2. [용어2]
원어: [원어2] | 분야: [활용 분야2]

Q3. [용어3]
원어: [원어3] | 분야: [활용 분야3]

⏸ 잠깐, 뜻을 떠올려보세요!
아래 메시지에서 정답을 확인하세요 👇
```

**메시지 2 — 정답 3개 묶음:**
```
━━━━━━━━━━━━━━━━━━
✅ 오늘의 정답 공개
━━━━━━━━━━━━━━━━━━

1️⃣ [용어1]
의미: [한 줄 정의1]
예시: "[실무 예시1]"
연관: [연관 용어1]

─────────────────
2️⃣ [용어2]
의미: [한 줄 정의2]
예시: "[실무 예시2]"
연관: [연관 용어2]

─────────────────
3️⃣ [용어3]
의미: [한 줄 정의3]
예시: "[실무 예시3]"
연관: [연관 용어3]
━━━━━━━━━━━━━━━━━━
다음 복습: [다음 복습 예정일] 👋
```

3개 용어 발송 완료 후 Notion을 업데이트합니다:
- 마지막 복습일 → 오늘
- 복습 횟수 → +1
- 다음 복습 예정일 → 아래 간격 표에 따라 자동 계산
- 숙지 상태 → '복습 중'으로 변경 (처음 복습이면)

---

### 5. 복습 간격 자동 계산

퀴즈 발송 시(또는 [정답확인] 후) Notion을 업데이트할 때, 현재 복습 횟수를 기준으로 다음 복습 예정일을 자동 계산합니다.

**기본 복습 간격표:**
| 복습 횟수 | 다음 복습까지 |
|----------|-------------|
| 0 → 1회차 | 1일 후 |
| 1 → 2회차 | 3일 후 |
| 2 → 3회차 | 7일 후 |
| 3 → 4회차 | 14일 후 |
| 4회 이상 | 30일 후 |

별도 응답이 없어도 퀴즈가 발송될 때마다 자동으로 간격이 늘어납니다.

---

### 6. 복습 스케줄 등록

`mcp__scheduled-tasks__create_scheduled_task`로 매일 오전 8:30 스케줄을 등록합니다.

스케줄 프롬프트:
```
업무 용어 학습 단어장(Notion)에서 오늘 복습할 용어 3개를 선정해주세요.
선정 기준: 숙지 완료 용어 제외, 다음 복습 예정일 기준 오늘 이전 → 오늘 → 복습 횟수 적은 순 → 등록일 오래된 순.
카카오톡 나에게 보내기(MemoChat)로 질문 3개 묶음 메시지 1개, 정답 3개 묶음 메시지 1개, 총 2개 메시지만 발송하세요.
발송 후 해당 용어들의 마지막 복습일을 오늘 날짜로 업데이트하세요.
```

---

### 7. 현황 조회

"미숙지 용어", "이번 주 복습 예정", "복습 중인 용어" 등 요청 시:
- `notion-query-database-view`로 조건에 맞는 용어 조회
- 표 형식으로 정리하여 답변

---

## 주의사항

- 데이터베이스를 찾을 때 "업무 용어 학습 단어장"을 정확히 검색합니다. 이름이 비슷한 여러 DB가 있으면 사용자에게 확인합니다.
- 카카오톡은 반드시 2개 메시지만 발송합니다: 첫 번째는 질문 3개 묶음, 두 번째는 정답 3개 묶음. 용어마다 따로 보내지 마세요.
- 숙지 완료 용어는 복습 대상에서 완전히 제외합니다.
- 용어 설명은 항상 실무 초급자 기준, 업계 관용 표현도 함께 설명합니다.
