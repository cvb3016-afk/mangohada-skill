Gmail 받은편지함의 안 읽은 메일을 지금 바로 확인해서 요약하고 Slack으로 전송한다.
(매일 오전 9:10 / 오후 3:00에 자동 실행되는 `gmail-slack-digest-morning`/`gmail-slack-digest-afternoon`과 같은 로직과 상태 파일을 공유하는 수동 버전이다. 이 둘은 그대로 유지하고, 이 스킬은 사용자가 그 사이에 즉시 확인하고 싶을 때 직접 호출한다.)

## 실행 절차

1. Gmail MCP(`search_threads`)로 안 읽은 메일 전체 조회 (`is:unread in:inbox`)
2. 상태 파일 `~/.gmail_slack_digest_state.json`을 읽어, 이미 전송한 thread ID는 건너뜀
3. 아래 이메일은 제외한다:
   - 뉴스레터, 구독 이메일 (수신 거부 링크 포함)
   - 쇼핑몰 프로모션, 할인, 이벤트
   - SNS 알림 (카카오, 인스타그램, 페이스북, 링크드인 등)
   - 자동 마케팅·홍보 메일 (no-reply 발신)
   - 시스템 자동 알림 (배송, 앱 알림 등 단순 정보)
4. 남은 이메일은 인사말·서명·홍보 문구 제거 후 핵심 내용 3~5줄로 한국어 요약
5. 회신 요청·일정 변경·계약·견적·결제 관련 메일은 🔴 우선순위 높음 표시
6. Slack MCP로 사용자 본인(user_id: U0BAXNG764F)에게 DM 전송. 형식:

```
📧 *이메일 요약* (N건) — M월 DD일 HH:MM 기준 (수동 확인)

━━━━━━━━━━━━━━━━━━━━━━
🔴 [우선순위 높음]  ← 해당 시만
*보낸사람:* 이름 <email>
*제목:* 제목
*수신시간:* YYYY-MM-DD HH:MM

*핵심내용:*
• 요약 1
• 요약 2
• 요약 3

*첨부파일:* 파일명 또는 없음
*원문 링크:* <https://mail.google.com/mail/u/0/#inbox/THREAD_ID|Gmail에서 보기>
━━━━━━━━━━━━━━━━━━━━━━
```

7. 전송 후 `~/.gmail_slack_digest_state.json` 업데이트 (last_run 시각 + 전송한 thread ID 추가, 7일 지난 ID는 삭제)
8. 보낼 메일이 없으면 Slack 메시지는 보내지 않고, 사용자에게 "새로 온 메일 없어요"라고 짧게 답한다
